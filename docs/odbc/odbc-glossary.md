# <a name="odbc-glossary"></a>ODBC 詞彙
## <a name="a"></a>A  
 **access plan**  
 所產生之計畫的資料庫引擎執行 SQL 陳述式。 從 c 這類第三代語言編譯的可執行程式碼的對等項目  
  
 **彙總函式**  
 從一組值，通常會搭配產生的單一值的函式**GROUP BY**並**HAVING**子句。 彙總函式包含**AVG**，**計數**，**最大**， **MIN**，以及**總和**。 也稱為*設定函式*。 *另請參閱*純量函式。  
  
 **ANSI**  
 American National Standards Institute。 ODBC API 是以 ANSI 呼叫層級介面為基礎。  
  
 **APD**  
 *請參閱*應用程式參數描述項 (APD)。  
  
 **API**  
 應用程式開發介面。 一組應用程式用來要求及推行較低層級服務常式。 ODBC API 會組成的 ODBC 函式。  
  
 **application**  
 呼叫 ODBC API 中的函式可執行程式。  
  
 **應用程式參數描述項 (APD)**  
 描述在應用程式指定任何轉換之前的 SQL 陳述式中使用的動態參數描述元。  
  
 **應用程式資料列描述項 (ARD)**  
 表示資料行中繼資料和應用程式的緩衝區，描述下列應用程式指定任何資料轉換的資料列中的資料描述元。  
  
 **ARD**  
 *請參閱*應用程式資料列描述項 (ARD)。  
  
 **auto-commit mode**  
 在其中交易被認可在執行之後，立即交易認可模式。  
  
## <a name="b"></a>B  
 **行為變更**  
 在特定的功能，從 ODBC 3 中變更 *.x* ODBC 2 的行為。*x*行為，或反之亦然。 因變更 SQL_ATTR_ODBC_VERSION 環境屬性。  
  
 **二進位大型物件 (BLOB)**  
 透過特定數目的位元組為單位，例如 255 任何二進位資料。 通常會更長。 這類資料通常會傳送至，並從組件中的資料來源擷取。 也稱為*長資料*。  
  
 **binding**  
 作為動詞使用，結果集中的資料行或 SQL 陳述式中的參數關聯的應用程式變數中的動作。 為名詞，關聯。  
  
 **binding offset**  
 值，所有繫結資料行或參數的資料，產生新位址加入至資料緩衝區位址和長度/指標緩衝區位址。  
  
 **區塊資料指標**  
 資料指標能夠一次提取一個以上的資料列的資料。  
  
 **buffer**  
 一種用來在應用程式和驅動程式之間傳遞資料的應用程式記憶體中。 緩衝區通常會成對出現：*資料緩衝區*並*的資料長度的緩衝區*。  
  
 **byte**  
 8 位元或一個八位元。 *另請參閱*八位元。  
  
## <a name="c"></a>c  
 **C 資料類型**  
 C 程式中，在此情況下應用程式中的變數資料類型。  
  
 **catalog**  
 描述資料庫的圖形的系統資料表在資料庫中的集合。 也稱為*結構描述*或是*資料字典*。  
  
 **目錄函式**  
 ODBC 函數，用來擷取資料庫的目錄中的資訊。  
  
 **CLI**  
 *請參閱*API。  
  
 **client/server**  
 資料庫存取策略，在其中一個或多個用戶端，請存取透過伺服器的資料。 用戶端在伺服器控制項資料庫存取時，通常會實作使用者介面。  
  
 **column**  
 資料列中的資訊的單一項目容器。 也稱為*欄位*。  
  
 **commit**  
 若要永久變更在交易中。  
  
 **concurrency**  
 在此同時存取相同的資料超過一筆交易的能力。  
  
 **一致性層級**  
 一組獨立的驅動程式或資料來源所支援的功能。 ODBC 定義 API 的一致性層級，以及 SQL 一致性層級。  
  
 **connection**  
 驅動程式和資料來源的特定執行個體。  
  
 **連接瀏覽**  
 正在搜尋資料來源，以連接到的網路。 連接瀏覽，可能需要幾個步驟。 例如，使用者可能會先瀏覽網路的伺服器，然後瀏覽特定資料庫的伺服器。  
  
 **連接控制代碼**  
 包含連線的相關資訊的資料結構的控制代碼。  
  
 **目前資料列**  
 目前資料指標所指向的資料列。 目前的資料列定位的作業運作。  
  
 **cursor**  
 一種應用程式傳回的資料列的軟體。 可能為名的終端機; 的電腦上的閃爍游標就像該資料指標會指出目前的位置，在畫面上，結果集的資料指標會指出結果集內的目前位置。  
  
## <a name="d"></a>D  
 **資料緩衝區**  
 緩衝區，用來傳遞資料。 通常相關聯的資料緩衝區已*的資料長度的緩衝區*。  
  
 **資料字典**  
 *請參閱*目錄。  
  
 **資料長度的緩衝區**  
 用來傳遞中對應值的長度的緩衝區*資料緩衝區*。 資料長度的緩衝區，也會將指標，例如資料值是 null 結束。  
  
 **資料來源**  
 （如果有的話），使用者想要存取其相關聯的作業系統、 DBMS 和網路平台的資料。  
  
 **資料類型**  
 一種資料型別。 ODBC 定義 C 」 和 「 SQL 資料類型。 *另請參閱*類型指標。  
  
 **data-at-execution column**  
 資料傳送後的資料行**SQLSetPos**呼叫。 如此命名是因為資料會在執行時間，而不是放在緩衝區中資料列集。 在執行階段組件通常是傳送長資料。  
  
 **data-at-execution parameter**  
 資料傳送後的參數**SQLExecute**或是**SQLExecDirect**呼叫。 如此命名是因為將資料傳送時的 SQL 陳述式執行而不是放在參數緩衝區。 在執行階段組件通常是傳送長資料。  
  
 **資料庫**  
 離散的在 DBMS 中的資料集合。 也 DBMS。  
  
 **資料庫引擎**  
 DBMS，以剖析和執行 SQL 陳述式並存取實體的資料中的軟體。  
  
 **DBMS**  
 資料庫管理系統。 實體資料庫與使用者之間的軟體層。 DBMS 會管理資料庫的所有存取。  
  
 **以 DBMS 為基礎的驅動程式**  
 驅動程式，可透過獨立的資料庫引擎存取實體的資料。  
  
 **DDL**  
 資料定義語言。 這些陳述式在 SQL 中定義，而不是操作資料。 例如， **CREATE TABLE**， **CREATE INDEX**，**授與**，以及**撤銷**。  
  
 **分隔的識別碼**  
 括住識別碼引號字元，因此它可以包含特殊字元，或符合關鍵字 (也稱為 quoted identifier) 的識別碼。  
  
 **descriptor**  
 資料結構，其中含有資料行的資料或將動態參數的相關資訊。 未定義的描述元的實體表示法應用程式獲得描述元的直接存取，只是藉由呼叫 ODBC 函數具有描述元控制代碼操作其欄位。  
  
 **桌面資料庫**  
 設計在個人電腦上執行 DBMS。 一般而言，這些 Dbms 不提供獨立的資料庫引擎，而必須透過檔案為基礎的驅動程式存取。 這些驅動程式中的引擎通常較低 SQL 和交易的支援。 比方說，dBASE、 Paradox、 Btrieve 或 Microsoft® FoxPro®。  
  
 **diagnostic**  
 包含所呼叫的最後一個函式的診斷資訊的記錄會使用特定的控制代碼。 診斷記錄會與環境、 連接、 陳述式，以及描述項控制代碼相關聯。  
  
 **DML**  
 資料操作語言。 這些陳述式在 SQL 中操作，而不是資料定義。 例如，**插入**，**更新**，**刪除**，以及**選取**。  
  
 **driver**  
 常式的程式庫公開在 ODBC API 函式。 驅動程式特有的單一 DBMS。  
  
 **驅動程式管理員**  
 管理驅動程式應用程式的存取權，例行的程式庫。 驅動程式管理員載入和卸載 （或連接到和從中斷連線） 驅動程式並將傳遞至正確的驅動程式的 ODBC 函數呼叫。  
  
 **驅動程式安裝程式 DLL**  
 包含特定驅動程式安裝和設定函式的 DLL。  
  
 **dynamic cursor**  
 可捲動資料指標能夠偵測更新、 刪除或插入結果集中的資料列。  
  
 **dynamic SQL**  
 在 SQL 陳述式所建立，並在執行階段編譯的內嵌 SQL 型別。 *另請參閱*靜態 SQL。  
  
## <a name="e"></a>E  
 **內嵌的 SQL**  
 直接在另一種語言，例如 COBOL 或 C.ODBC 所撰寫的程式中包含的 SQL 陳述式不會使用內嵌的 SQL。 *另請參閱*靜態 SQL*和*動態 SQL。  
  
 **environment**  
 全域內容中存取資料;與環境相關聯是為全球性的本質，例如一份該環境中的所有連接的任何資訊。  
  
 **環境控制代碼**  
 包含環境的相關資訊的資料結構的控制代碼。  
  
 **逸出子句**  
 SQL 陳述式中的子句。  
  
 **execute**  
 若要執行的 SQL 陳述式。  
  
## <a name="f"></a>F  
 **資料指標**  
 *請參閱*區塊資料指標。  
  
 **fetch**  
 擷取一或多個資料列，從結果集。  
  
 **field**  
 *請參閱*資料行。  
  
 **檔案為基礎的驅動程式**  
 驅動程式，可直接存取實體的資料。 在此情況下，驅動程式會包含資料庫引擎，並做為驅動程式和資料來源。  
  
 **檔案資料來源**  
 資料來源的連接資訊會儲存在.dsn 檔案。  
  
 **外部索引鍵**  
 在資料表中符合另一個資料表中的主索引鍵的資料行或資料行。  
  
 **順向資料指標**  
 資料指標，可以只繼續透過結果集，通常會擷取一次只有一個資料列。 大部分的關聯式資料庫支援只順向資料指標。  
  
## <a name="h"></a>H  
 **handle**  
 值，這個值可唯一識別項目，例如檔案或資料結構。 控制代碼是僅有意義的軟體，會建立和使用它們，但傳遞的其他軟體以找出項目。 ODBC 定義環境、 連接、 陳述式，以及描述項控制的代碼。  
  
## <a name="i"></a>I  
 **實作參數描述項 (IPD)**  
 描述應用程式指定任何轉換後的 SQL 陳述式中使用的動態參數描述元。  
  
 **實作資料列描述項 (IRD)**  
 描述在應用程式指定任何轉換之前的資料列描述項。  
  
 **安裝程式 DLL**  
 安裝 ODBC 元件及設定資料來源的 DLL。  
  
 **Integrity Enhancement Facility**  
 SQL 子集設計用來維護資料庫的完整性。  
  
 **介面一致性層級**  
 驅動程式; 所支援的 ODBC 3.7 介面層級可以是核心、 層級 1 或層級 2。  
  
 **interoperability**  
 若要存取不同的 Dbms 中的資料時，使用相同的程式碼的應用程式的能力。  
  
 **IPD**  
 *請參閱*實作參數描述項 (IPD)。  
  
 **IRD**  
 *請參閱*實作資料列描述項 (IRD)。  
  
 **ISO/IEC**  
 國際標準組織/國際電子電機委員會。 ODBC API 是以 ISO/IEC 呼叫層級介面為基礎。  
  
## <a name="j"></a>J  
 **join**  
 藉由比對中的值會連結兩個或多個資料表中的資料列的關聯式資料庫中的作業指定的資料行。  
  
## <a name="k"></a>K  
 **key**  
 其值會識別資料列的資料行或資料行。 *另請參閱*外部索引鍵*和*主索引鍵。  
  
 **keyset**  
 一組混合，或按一下 索引鍵集驅動資料指標用來重新提取資料列的索引鍵。  
  
 **keyset-driven cursor**  
 使用索引鍵集，偵測到更新和刪除資料列可捲動資料指標。  
  
## <a name="l"></a>L  
 **literal**  
 SQL 陳述式中的實際資料值的字元表示法。  
  
 **locking**  
 程序，DBMS 將存取限制在多使用者環境中的資料列。 DBMS 通常設定一個位元上的資料列或 [實體] 頁面，包含表示的資料列的資料列或頁面鎖定。  
  
 **長資料**  
 超過特定長度，例如 255 個位元組或字元任何二進位或字元資料。 通常會更長。 這類資料通常會傳送至，並從組件中的資料來源擷取。 也稱為*BLOB*s 或*CLOB*s。  
  
## <a name="m"></a>M  
 **機器的資料來源**  
 資料來源的連接資訊會儲存在系統 （例如登錄）。  
  
 **manual-commit mode**  
 交易認可的模式下，在其中交易必須明確地認可藉由呼叫**SQLTransact**。  
  
 **metadata**  
 描述 SQL 陳述式中的參數或結果集中的資料行的資料。 比方說，資料類型、 位元組長度和精確度的參數。  
  
 **多層式架構的驅動程式**  
 *請參閱*以 DBMS 為基礎的驅動程式。  
  
## <a name="n"></a>N  
 **NULL 值**  
 有沒有明確指派的值。 特別是，NULL 值與不同的零或空白。  
  
## <a name="o"></a>O  
 **octet**  
 8 位元或一個位元組。 *另請參閱*位元組。  
  
 **八位元長度**  
 在八位元的緩衝區或它所包含的資料長度。  
  
 **ODBC**  
 開放式資料庫連接。 定義一組標準的常式與應用程式可以存取資料來源中的資料的 API 規格。  
  
 **ODBC 管理員**  
 安裝程式來設定資料來源的 DLL 會呼叫可執行程式。  
  
 開啟群組  
 發行標準公司。 特別是，它會將發佈 SQL 存取群組 (SAG) 標準。  
  
 **開放式並行存取**  
 若要提升並行數目不會鎖定資料列是策略。 相反地，它們被更新或刪除之前，資料指標會檢查以查看它們是否已經過變更自上次讀取後。 如果是的話，就無法更新或刪除。 *另請參閱*封閉式並行存取。  
  
 **外部聯結**  
 聯結中，會傳回相符的和不符合資料列。 不符合資料列中不相符的資料表中的所有資料行的值會設為 NULL。  
  
 **owner**  
 資料表的擁有者。  
  
## <a name="p"></a>P  
 **parameter**  
 中的 SQL 陳述式，以參數標記或問號 （？） 標示的變數。 參數繫結至應用程式變數與陳述式執行時擷取其值。  
  
 **parameter descriptor**  
 說明中的 SQL 陳述式，請指定應用程式 （應用程式參數描述項或 APD） 或任何應用程式 （實作所指定的轉換後的任何轉換之前使用的執行階段參數描述元參數描述項或 IPD）。  
  
 **參數作業陣列**  
 陣列，包含應用程式可以設定，表示應該忽略中的對應參數的值**SQLExecDirect**或是**SQLExecute**作業。  
  
 **參數狀態陣列**  
 陣列，包含參數的狀態之後呼叫**SQLExecDirect**或是**SQLExecute**。  
  
 **封閉式並行存取**  
 實作可序列化能力，使其他交易就無法變更這些資料列鎖定策略。 *另請參閱*開放式並行存取*和*可序列化能力。  
  
 **定位的作業**  
 在目前的資料列所做的任何作業。 例如，定位 update 和 delete 陳述式， **SQLGetData**，並**SQLSetPos**。  
  
 **定位的 update 陳述式**  
 SQL 陳述式來更新目前的資料列中的值。  
  
 **定位的 delete 陳述式**  
 SQL 陳述式用來刪除目前的資料列。  
  
 **prepare**  
 若要編譯 SQL 陳述式。 準備 SQL 陳述式時，會建立存取計劃。  
  
 **主索引鍵**  
 可唯一識別資料表中的資料列的資料行或資料行。  
  
 **procedure**  
 一或多個群組先行編譯為具名物件的資料庫中儲存的 SQL 陳述式。  
  
 **程序資料行**  
 在程序呼叫中，程序中或在程序所建立的結果集中的資料行所傳回的值為引數。  
  
## <a name="q"></a>Q  
 **qualifier**  
 包含一或多個資料表的資料庫。  
  
 **query**  
 SQL 陳述式。 有時可用來表示**選取**陳述式。  
  
 **引號識別碼**  
 識別項，使其可以包含特殊字元或關鍵字 （在 SQL-92 作為分隔識別碼中也稱為） 相符住識別項的引號字元。  
  
## <a name="r"></a>R  
 **radix**  
 基底的數字系統。 通常是 2 或 10。  
  
 **record**  
 *請參閱*資料列。  
  
 **結果集**  
 執行所建立的資料列集**選取**陳述式。  
  
 **傳回碼**  
 ODBC 函數所傳回的值。  
  
 **回復**  
 若要傳回其原始狀態的交易變更的值。  
  
 **row**  
 一組相關的資料行描述特定的實體。 也稱為*記錄*。  
  
 **資料列描述項**  
 描述結果集，請指定應用程式 （實作資料列描述項或 IRD） 或應用程式 （應用程式的資料列描述項或 ARD） 指定任何轉換後的任何轉換之前的資料行描述元。  
  
 **資料列作業陣列**  
 陣列，包含應用程式可以設定，表示應該忽略中的對應的資料列的值**SQLSetPos**作業。  
  
 **資料列狀態陣列**  
 陣列，包含資料列的狀態之後呼叫**SQLFetch**， **SQLFetchScroll**，或**SQLSetPos**。  
  
 **rowset**  
 區塊資料指標所傳回單一擷取中的資料列集。  
  
 **資料列集的緩衝區**  
 緩衝區繫結至資料行的結果集和中，會傳回整個資料列集的資料。  
  
## <a name="s"></a>S  
 **SAG**  
 *請參閱*SQL 存取群組 (SAG)。  
  
 **純量函式**  
 從單一值產生單一值的函式。 例如，字元資料的大小寫變更的函式。  
  
 **schema**  
 *請參閱*目錄。  
  
 **可捲動資料指標**  
 可以向前或向後移動透過結果集資料指標。  
  
 **serializability**  
 是否同時執行兩個交易產生這些交易的序列 （或連續） 執行相同的結果。 可序列化的交易，才能維護資料庫的完整性。  
  
 **伺服器資料庫**  
 設計在用戶端/伺服器環境中執行 DBMS。 這些 Dbms 提供獨立的資料庫引擎，可提供豐富的支援 SQL 和交易。 它們是透過以 DBMS 為基礎的驅動程式進行存取。 例如，Oracle、 Informix、 DB/2 或 Microsoft® SQL Server。  
  
 **set 函式**  
 *請參閱*彙總函式。  
  
 **setup DLL**  
 *請參閱*驅動程式安裝 DLL*和*轉譯程式安裝程式 DLL。  
  
 **單層式架構的驅動程式**  
 *請參閱*檔案為基礎的驅動程式。  
  
 **SQL**  
 結構化的查詢語言。 若要查詢，關聯式資料庫所用的語言更新和管理資料。  
  
 **SQL 存取群組 (SAG)**  
 公司關心 SQL Dbms 同業協會。 開啟群組的呼叫層級介面根據原本由 SQL Access 群組的工作。  
  
 **SQL 一致性層級**  
 驅動程式; 所支援的 SQL-92 語法層級可以是項目、 FIPS Transitional、 中級者、 或全文。  
  
 **SQL 資料類型**  
 資料來源中儲存的資料行或參數，因為它的資料類型。  
  
 **SQLSTATE**  
 五個字元的值，指出特定錯誤。  
  
 **SQL 陳述式**  
 在 SQL 中以關鍵字開頭，且完整地描述了所要採取的動作的完整片語。 例如，選取 * 從訂單。 SQL 陳述式不應混淆陳述式。  
  
 **state**  
 項目定義完善的條件。 例如，連線有七個狀態，包括未配置、 配置、 已連線，且需要的資料。 只有項目處於特定狀態時，就可以完成特定作業。 比方說時，才在配置的狀態，而且，如範例中，處於連線狀態時，可以釋放連接。  
  
 **狀態轉換**  
 從某個狀態到另一個項目的移動。 ODBC 定義的嚴格的狀態轉換的環境、 連接和陳述式。  
  
 **陳述式**  
 在 SQL 陳述式相關的所有資訊的容器。 陳述式不應該使用 SQL 陳述式相混淆。  
  
 **陳述式控制代碼**  
 包含陳述式的相關資訊的資料結構的控制代碼。  
  
 **靜態資料指標**  
 無法偵測到更新，可捲動資料指標刪除，或插入結果集內。 通常被藉由建立結果集的複本。  
  
 **static SQL**  
 一種內嵌的 SQL 陳述式會在 SQL 中硬式編碼，並編譯編譯程式的其餘部分時。 *另請參閱*動態 SQL。  
  
 **預存程序**  
 *請參閱*程序。  
  
## <a name="t"></a>T  
 **table**  
 資料列集合。  
  
 **thunking**  
 32 位元 ODBC 驅動程式搭配使用 16 位元應用程式時，轉換的 16 位元位址以 32 位元位址，或反之亦然。  
  
 **transaction**  
 不可部分完成的工作單位。 在交易中的工作必須完成整個;如果交易的任何部分失敗，整個交易就會失敗。  
  
 **交易隔離**  
 隔離的所有其他交易在影響的一筆交易的動作。  
  
 **交易隔離等級**  
 交易隔離程度的量值。 有五個交易隔離等級：讀取未認可，讀取認可、 可重複讀取可序列化，以及版本控制。  
  
 **translator DLL**  
 DLL 會用來平移設為另一個字元的資料。  
  
 **轉譯程式安裝程式 DLL**  
 包含轉譯器特有的安裝和設定的函式的 DLL。  
  
 **兩階段交易認可**  
 認可分散式的交易中兩個階段的程序。 在第一個階段中，交易處理器會檢查所有組件的交易可以認可。 在第二個階段中，所有組件的交易已認可。 如果交易的任何一部分的第一個階段中指出它無法認可，不會發生在第二個階段。 ODBC 不支援兩階段交易認可。  
  
 **類型指標**  
 整數值傳遞，或傳回 ODBC 函式，以指出應用程式變數、 參數或資料行的資料類型。 ODBC 定義 C 和 SQL 資料類型的類型的指標。  
  
## <a name="v"></a>V  
 **view**  
 另一種查看一或多個資料表中的資料。 通常以一或多個資料表的資料行的子集來建立檢視。 在 ODBC 中，檢視則是一般資料表與對等項目。
