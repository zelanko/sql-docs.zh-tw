# <a name="types-of-descriptors"></a>描述項的類型
描述元用來描述下列其中一項：  
  
-   一組零或多個參數。 參數描述項可用來描述：  
  
    -   *應用程式參數緩衝區*其中包含其中一個輸入動態引數所設定的應用程式或後面執行的動態引數輸出**呼叫**SQL 陳述式。  
  
    -   *實作參數緩衝區*。 為輸入的動態引數，這會包含應用程式參數緩衝區中的相同引數之後，應用程式可能會指定任何資料轉換。 輸出動態引數，這包含應用程式可以指定任何資料轉換前的傳回引數。  
  
     為輸入的動態引數，應用程式必須對應用程式參數描述項，然後再執行包含動態參數標記的任何 SQL 陳述式。 應用程式可以針對輸入和輸出動態引數，指定在實作參數描述項中的不同資料類型，以達到資料轉換。  
  
-   資料庫資料的單一資料列。 資料列描述項可用來描述：  
  
    -   *實作資料列緩衝區，* 其中包含從資料庫的資料列。 （這些緩衝區在概念上包含的資料寫入或讀取資料庫。 不過，未指定資料庫資料的預存的形式。 資料庫無法執行其他轉換其表單實作緩衝區中的資料。）  
  
    -   *應用程式的資料列緩衝區*，呈現給應用程式，並遵循應用程式可以指定任何資料轉換，其中會包含一個資料列。  
  
     應用程式上運作的應用程式的資料列描述項在任何情況下必須在應用程式變數中會出現在資料庫中的資料行資料。 若要達到資料行資料的資料轉換，應用程式可以指定不同的資料類型所實作的資料列描述項。  
  
 下表摘要說明描述項類型。  
  
|緩衝區類型|資料列|動態參數|  
|-----------------|----------|------------------------|  
|**應用程式的緩衝區**|應用程式資料列描述項 (ARD)|應用程式參數描述項 (APD)|  
|**實作緩衝區**|實作資料列描述項 (IRD)|實作參數描述項 (IPD)|  
  
 參數或資料列緩衝區中，如果應用程式指定不同的資料類型中對應記錄的實作和應用程式的描述元，驅動程式會執行資料轉換時它會使用描述元。 比方說，它可能會將數值和日期時間值轉換成字元字串格式。 (針對有效的轉換，請參閱[附錄 d:資料型別](../../../odbc/reference/appendixes/appendix-d-data-types.md)。)  
  
 描述元可以執行不同的角色。 不同的陳述式可以共用應用程式明確配置任何描述項。 在單一陳述式的資料列描述項可以做為另一個陳述式中的參數描述項。  
  
 一律知道給定的描述項是否為應用程式描述項或實作描述項，即使描述元尚未使用的資料庫作業。 實作隱含配置描述元，實作會記錄預先定義的資料列，相對於陳述式控制代碼。 藉由呼叫的應用程式會配置任何描述元**SQLAllocHandle**是應用程式描述項。
