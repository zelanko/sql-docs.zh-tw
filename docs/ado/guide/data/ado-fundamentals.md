# <a name="ado-fundamentals"></a>ADO 基本概念
ADO 為開發人員提供強大的邏輯物件模型，可透過 OLE DB 系統介面，以程式設計方式存取、編輯及更新各種資料來源中的資料。 ADO 最常見的用法是查詢關係資料庫中的資料表或資料表、取出並在應用程式中顯示結果，而且可能讓使用者進行和儲存資料的變更。 其他工作包含下列各項：  
  
-   使用 SQL 查詢資料庫並顯示結果。  
  
-   透過網際網路存取檔案存放區中的資訊。  
  
-   在電子郵件系統中操作訊息和資料夾。  
  
-   將資料庫中的資料儲存至 XML 檔案。  
  
-   執行以 XML 描述的命令並取出 XML 資料流程。  
  
-   將資料儲存到二進位或 XML 資料流程中。  
  
-   允許使用者檢查及變更資料庫資料表中的資料。  
  
-   建立和重複使用參數化資料庫命令。  
  
-   執行預存程式。  
  
-   以動態方式建立名為 **記錄集**的彈性結構，以保存、流覽和運算元據。  
  
-   執行交易式資料庫作業。  
  
-   根據執行時間準則篩選和排序資料庫資訊的本機複本。  
  
-   從資料庫建立和操作階層式結果。  
  
-   將資料庫欄位系結至可感知資料的元件。  
  
-   建立遠端、中斷連接的 **記錄集**。  
  
 ADO 會公開各種不同的選項和設定，以提供這種彈性。 因此，請務必採取一種方式來學習如何在應用程式中使用 ADO，將每個目標細分成可管理的部分。  
  
 大部分使用 ADO 的應用程式都牽涉到四個主要作業：取得資料、檢查資料、編輯資料，以及更新資料。 本節稍後會更詳細地檢查這些作業。
