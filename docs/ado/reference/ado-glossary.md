# <a name="ado-glossary"></a>ADO 詞彙
本主題定義適用於 ADO 的詞彙。  
  
## <a name="a"></a>只有在次要複本設定成手動容錯移轉模式，而且至少一個次要複本目前與主要複本 SYNCHRONIZED 時，  
 絕對 URL  
 完整的 URL，在網際網路或內部網路上指定之資源的所在位置。 另請參閱*URL*和*相對 URL*。  
  
 ActiveX 控制項  
 自我登錄同處理序通常會有視覺項目在設計階段或執行的階段的 COM 元件。 ActiveX 控制項也有一個主動式文件容器，例如 Microsoft Internet Explorer 與通訊的能力。  
  
 ADISAPI （進階資料網際網路伺服器應用程式開發介面）  
 提供剖析的 ISAPI DLL、 Automation 控制項、 封送處理資料錄集，以及 MIME 封裝。 ADISAPI 元件適用於透過網際網路資訊服務 (IIS) 提供的 API。 另請參閱*ISAPI*。  
  
 彙總函式 (aggregate function)  
 在查詢中，例如 COUNT、 AVG 或計算資料行的資料表中使用的所有資料列的值的標準差的函式。 在撰寫運算式和程式設計中，您可以使用 SQL 彙總函式 （包括三個以上所列） 和網域彙總函式來判斷各種統計資料。  
  
 alias  
 您提供給資料行或運算式中的 SQL SELECT 陳述式，通常較短或更有意義的替代名稱。 例如，BobSales 是下列的 SELECT 陳述式中的別名: 「 為從 SalesDB BobSales 選取將銷售 」。 別名可以用來動態指派至 DataControl 物件上的控制項繫結的資料行。  
  
 Apartment 執行緒  
 COM 執行緒的模型物件的所有呼叫都發生在一個執行緒。 在 apartment 執行緒，COM 會同步處理，並將呼叫封送處理。 另請參閱*COMmddefcom*。  
  
 非同步作業  
 控制權傳回給呼叫程式，而不需等待作業完成的作業。 作業已完成之前，會繼續執行程式碼。 另請參閱*同步作業*。  
  
## <a name="b"></a>B  
 繫結項目  
 資料表中的欄位和變數之間的對應。 ADO Visual c + + 擴充功能中**資料錄集**欄位會對應至 C/c + + 變數。  
  
 位元遮罩  
 數字的值適用於搭配其他數值的位元值比較通常加上旗標參數或傳回值中的選項。 通常這項比較會完成的位元邏輯運算子，例如**和**和**或者**在 Visual Basic 中**&**和**&#124;** c + + 中。  
  
 例如，ADO **FieldAttributeEnum**值可用來當作位元遮罩來決定欄位的屬性。 假設您想要判斷是否可更新的欄位。 您可以測試這個，與 Visual Basic 中的下列運算式：`Field.Attributes AND adFldUpdatable`  
  
 如果結果為 TRUE，此欄位是可更新。  
  
 書籤 (bookmark)  
 標記可唯一識別資料列集內的資料列，以便使用者可以快速瀏覽到它。  
  
 商務物件  
 執行一組定義的作業，例如資料驗證或商務規則邏輯的物件。 通常位於中介層商務物件。  
  
 商務規則  
 驗證的編輯、 登入驗證、 資料庫查閱、 原則和構成企業的方式進行商務交易的演算式轉換組合。 也稱為*商務邏輯*。  
  
## <a name="c"></a>C  
 計算的運算式  
 運算式不是常數，但其值取決於其他值。 要評估導出的運算式必須取得並計算從其他來源，通常是在其他欄位或資料列的值。  
  
 本文章節  
 從資料來源的資料列範圍參考。 在 ADO 中，章節通常是另一個的參考**資料錄集**。  
  
 章節資料行讓您定義*父子式*關聯性其中*父*是**資料錄集**包含章節資料行和*子*是**資料錄集**本章所表示。  
  
 章別名  
 參考附加至父代的資料行別名。  
  
 字元集 (character set)  
 一組字元，其值必須為數字的對應。 比方說，Unicode 是 16 位元字元集能夠編碼所有的已知的字元做為全球的字元編碼標準。  
  
 child  
 階層式關聯性的相依端。 子系是具有其上方的另一個節點的階層式結構中的節點 （更接近根）。 另請參閱*子別名*，*父子式關聯性*，*父*。  
  
 子別名  
 別名所參考之子系。 另請參閱*別名*，*子*。  
  
 CLSID （類別識別項）  
 通用唯一識別碼 (UUID) 識別 COM 元件。 每個 COM 元件會有 CLSID Windows 登錄中，因此其他應用程式可以載入。 另請參閱*ProgID*， *COM*。  
  
 用戶端層  
 通常從使用者，都會呈現至資料和輸入的處理序的分散式系統的邏輯層有時也稱為*前端*。 通常，用戶端層從根據輸入，伺服器要求資料，然後格式化並顯示結果。 另請參閱*中介層*，*資料來源層*，*分散式應用程式*。  
  
 COM （元件物件模型）  
 二進位的標準，可讓交互操作，無論他們所開發的語言，或它們所在的電腦上的網路環境中的物件。 以 COM 為基礎的技術包括 ActiveX 控制項、 自動化和連結和內嵌 (OLE) 的物件。 COM，允許物件來公開其功能給其他元件和主控件應用程式。 它會定義如何物件會公開本身和跨處理序和跨網路，此公開資訊的運作方式。 COM 也會定義物件的生命週期。  
  
 COM 元件  
 二進位檔案，例如.dll、.ocx 及某些.exe 檔案 — 所支援的 COM 標準提供物件。 這類檔案包含一或多個 class factory，COM 類別、 登錄項目機制、 載入程式碼，以及等等的程式碼。  
  
 比較運算子  
 運算子會比較兩個運算式，並傳回布林值。  
  
 可能表示為條件參數">"（大於）、"\<"（小於）、"="（等於）、"> ="（大於或等於），"< ="（小於或等於）、"<>"（不等於） 或 [讚] （模式比對）。  
  
 component  
 物件，封裝資料與程式碼，並提供一組正確地指定公開可用的服務。  
  
 複合檔案  
 COM 實作結構化檔案的儲存體。 複合檔案儲存個別物件的單一、 結構化檔案中的兩個主要元素組成： 儲存物件和資料流物件。 在一起，這些函式類似檔案系統的檔案內。  
  
 繫結在一起，一個實體檔案中的個別檔案的數目。 可以存取在複合檔案中每個個別檔案，如同它是單一的實體檔案。  
  
 常數  
 數值或字串值，不會變更。 具名的 ADO 列舉型別 （列舉常數） 可用於您的程式碼，而不是實際的值，例如**adUseClient**為的常數，其值為 3。 (Const adUseClient = 3)。 另請參閱*列舉*。  
  
 cursor  
 Database 元素可控制記錄瀏覽、 可更新性的資料，與其他使用者對資料庫所做的變更的可見性。  
  
## <a name="d"></a>D  
 資料繫結  
 物件或資料來源的應用程式的控制項之間建立關聯的程序。 與資料來源相關聯的控制項稱為*資料繫結控制項*。  
  
 資料繫結控制項的內容相關聯資料庫的值。 例如，繫結至方格控制項**資料錄集**物件可以是更新時中的資料列**資料錄集**會更新。 當新的值藉由擷取**資料錄集**，新值會顯示在方格中。  
  
 資料提供者 (data provider)  
 軟體資料公開至 ADO 應用程式的直接或透過服務提供者。 另請參閱服務提供者。  
  
 塑造資料  
 技術，其利用的正式語法 (稱為**圖形語言**) 來定義特殊**資料錄集**物件 (稱為*塑造資料錄集*)，其中不包含只要資料，但是也參考其他**資料錄集**物件和/或計算的值根據這些其他**資料錄集**物件。  
  
 資料來源層  
 代表執行 DBMS，例如 SQL Server 資料庫的電腦是分散式系統的邏輯層。 另請參閱*用戶端層*，*中介層*，*分散式應用程式*。  
  
 DCOM  
 網路通訊協定可讓 COM 元件，能夠透過網路彼此直接通訊。 另請參閱*COM*，*元件*。  
  
 DDL （資料定義語言）  
 這些陳述式在 SQL 中定義，而非在操作資料。 建立或修改與 DDL 的資料庫結構描述。 例如， **CREATE TABLE**， **CREATE INDEX**， **GRANT**，和**撤銷**SQL DDL 陳述式。  
  
 預設資料流  
 文字或二進位資料流 (由**資料流**物件) 相關聯**記錄**或**資料錄集**物件時使用特定的 OLE DB 提供者，例如網際網路發行的 Microsoft OLE DB 提供者。 預設資料流通常包含網站的根例如的 HTML 程式碼檔案的內容。  
  
 分散式應用程式  
 撰寫，以便處理可以透過網路分割多部電腦上的程式。 一般而言，分散式應用程式將資料分為簡報、 商務邏輯和資料存放區的圖層，或*層*。 另請參閱用戶端層、 中介層、 資料來源層。  
  
 已中斷連線的資料錄集  
 A**資料錄集**也不再有即時連接到伺服器的用戶端快取中的物件。 如果原始資料來源需要存取一次基於某些原因，例如更新的資料，則必須重新建立連線。 不過，集合、 屬性和方法的已中斷連線**資料錄集**仍可存取。  
  
 DML （資料操作語言）  
 操作，而非在定義中，資料在 SQL 中這些陳述式。 選取，修改 DML 與資料庫中的值。 例如，**插入**，**更新**，**刪除**，和**選取**SQL DML 陳述式。  
  
 文件的來源提供者  
 是特殊類別的管理資料夾和文件的提供者。 文件由當**記錄**物件或文件的資料夾由**資料錄集**物件，文件的來源提供者會填入這些物件與一組唯一的欄位，描述文件，而不是實際的文件本身的特性。 另請參閱資源記錄。  
  
 資料來源名稱 （資料來源名稱）  
 用來連接到特定 ODBC 資料庫應用程式的資訊集合。 ODBC 驅動程式管理員會使用這項資訊來建立資料庫的連接。 在檔案 （檔案 DSN） 或 Windows 登錄 （DSN 的電腦） 中，可以儲存 DSN。  
  
 動態屬性  
 此資料提供者或資料指標服務特有的屬性。 **屬性**自動填入這些物件的集合 （「 動態 」）。 連線到資料來源透過特定的資料提供者之前，物件會有任何動態屬性。 另請參閱資料提供者、 資料指標。  
  
## <a name="e"></a>E  
 列舉型別  
 具名常數的清單。 列舉的值不需要是唯一的。 但是每個值的名稱必須是列舉型別定義所在的範圍內唯一的。 在 ADO 中，列舉型別用於數字參數和傳回值、 加入至 ADO 程式碼的意義，以及可為開發人員之數字的值 （這可能會變更版本）。 例如，若要開啟靜態**資料錄集**，使用**adOpenStatic**列舉值： `Recordset.Open ,,adOpenStatic`  
  
 也稱為*列舉的常數*。 另請參閱*常數*。  
  
 event  
 物件，您可以撰寫程式碼來回應所辨識的動作。 命令執行、 交易完成、 資料錄集導覽和資料更新，以及其他動作可以產生事件。 另請參閱*事件處理常式*。  
  
 事件處理常式 (event handler)  
 事件處理常式是事件發生時執行的程式碼。 另請參閱事件。  
  
## <a name="h"></a>H  
 handler (處理常式)  
 所管理之常見且相當簡單的條件或作業，例如錯誤復原或資料管理的常式。  
  
 階層式資料錄集  
 A**資料錄集**包含另一個**資料錄集**。 另請參閱資料成形，章節。  
  
 如需詳細資訊，請參閱[存取階層式資料錄集中的資料列](../../ado/guide/data/accessing-rows-in-a-hierarchical-recordset.md)。  
  
 階層 (hierarchy)  
 一般情況下，階層是使用最上層等級的結構層級和從屬層級。 在 ADO 中，階層式**資料錄集**用來表示第一筆記錄之間的父子式關聯性。 在 ADO 中，而且**記錄**和**資料流**物件可以用來存取功能的階層樹狀結構，例如資料夾和文件。 ADO MD 也包含**階層**表示 OLAP cube 中維度的層級之間的關聯性物件。 另請參閱階層式資料錄集父子式關聯性、 章、 樹狀結構。  
  
## <a name="i-l"></a>I-L  
 ISAPI （網際網路伺服器應用程式開發介面）  
 一組為網際網路的伺服器，例如執行 Microsoft® 網際網路資訊服務 (IIS) Windows NT® Server/Windows 2000 Server 的函式。  
  
 索引鍵  
 資料行或資料表的資料行可唯一識別資料列，通常用來編製索引的資料表。  
  
## <a name="m"></a>M  
 封送處理  
 跨執行緒或處理序界限的封裝、 傳送和 unpackaging 介面方法參數的程序。  
  
 中介層  
 使用者介面或 Web 用戶端與資料庫之間的分散式系統中的邏輯圖層。 這通常是商務物件具現化的位置。 中介層是商務規則和產生並操作接收資訊的函式的集合。 他們完成透過商務規則，它可以經常變更，並因此封裝成實際分開的應用程式邏輯本身的元件。 也稱為*應用程式伺服器層*。 分散式應用程式、 用戶端層、 資料來源層，請參閱。  
  
 MIME （多用途網際網路郵件延伸標準）  
 網際網路通訊協定原本被開發跨異質網路、 電腦和電子郵件環境允許以豐富內容的電子郵件訊息的交換。 在實務上，MIME 也已採用並擴充非電子郵件應用程式。  
  
 MIME 是一種標準，可讓要發行和讀取在網際網路上的二進位資料。 使用二進位資料檔案的標頭包含的 MIME 類型的資料;這會通知用戶端程式 (Web 瀏覽器和郵件封裝執行個體)，它們必須以不同方式處理資料，比處理純文字。 例如，包含 JPEG 圖片的 Web 文件的標頭包含 JPEG 檔格式的具體的 MIME 類型。 這可讓瀏覽器顯示的檔案，利用其 JPEG 檢視器中，如果有的話。  
  
## <a name="n-o"></a>N-O  
 node  
 階層樹狀結構中的項目。 節點可能會是根或另一個節點的子系。 節點也可以是多個子系的父系。 另請參閱階層、 樹狀目錄、 根、 子系、 父系。  
  
 物件變數  
 包含對物件之參考的變數。 例如，`objCustomObject`是指向 CustomObject 類型的物件變數：`Set objCustomObject = CreateObject(adodb.Recordset)`  
  
 ODBC (開放式資料庫連接)  
 用來連接到各種資料來源的標準程式設計語言介面。 這通常是透過 [控制台]，其中指定資料來源名稱 (Dsn)，使用特定的 ODBC 驅動程式存取。  
  
 OLE DB  
 一組將資料從各種來源使用 COM 公開的介面 OLE DB 介面會提供應用程式統一存取各種資訊來源中所儲存的資料。 這些介面支援 DBMS 功能適用於資料來源，讓它共用它的資料的數量。 另請參閱 com。  
  
 開放式鎖定  
 一種鎖定類型，其中包含一或多個記錄，包括正在編輯的記錄的資料頁是與其他使用者無法使用正由更新記錄時，才**更新**方法，但已使用之前和之後若要呼叫**更新**。  
  
 使用開放式鎖定時**資料錄集**物件開啟**LockType**參數或屬性設定為**Adlockreadonly**或**Adlockpessimistic**。 請參閱也封閉式鎖定。  
  
 序數值  
 訂單中項目的數值的位置。 ADO 集合中第一個項目的序數值為零 (0)。 下一個項目是一 （1） 等等。  
  
## <a name="p"></a>P  
 參數化的命令  
 查詢或命令，可讓您執行命令之前，設定參數值。 例如，SQL 字串可以參數化 SQL 字串中內嵌參數標記 (所指定 '？ ' 字元)。 然後應用程式的每個參數指定值，並執行該命令。  
  
 父系 (parent)  
 在控制的一端階層式關聯性。 在階層式結構中，父代有階層中的正下方的一個或多個子節點。 另請參閱父別名、 父子式關聯性、 子系。  
  
 父別名  
 父代是指別名。 請參閱也別名，父代。  
  
 父子式關聯性  
 在其父代是更高版本，並直接相關聯的一個或多個子系的一個層級的階層式結構中的關聯性。 子系是一個較低的層級，必須有一個父代。 另請參閱父系、 子系。  
  
 封閉式鎖定  
 一種鎖定類型，其中包含一或多個記錄，包括正在編輯的記錄的頁面是以確保更新，將會進行其他使用者無法使用。 封閉式鎖定行為是由 OLE DB 提供者定義。 一般而言，會鎖定時編輯記錄，仍無法使用，直到**更新**方法已完成。  
  
 封閉式鎖定已啟用時**資料錄集**物件開啟**LockType**參數或屬性設定為**Locktype**。 請參閱也開放式鎖定。  
  
 集區  
 效能最佳化，根據使用的預先配置的資源，例如物件或資料庫連接的集合。 它是從比建立新的資源集區抽出現有資源的效率。  
  
 ProgID （程式設計識別項）  
 對應至 Windows 登錄的 COM 應用程式的唯一名稱。 ADO 連接的 ProgID 是"ADODB。連接 」。 另請參閱 CLSID、 com。  
  
 proxy  
 提供參數封送處理的特定介面的物件和用戶端來呼叫在不同的執行緒或另一個處理序在不同的執行環境中，執行這類的應用程式物件所需的通訊。 Proxy 會位於與用戶端，而且與對應的虛設常式位於與應用程式物件所呼叫之通訊。 另請參閱虛設常式。  
  
## <a name="r"></a>R  
 相對的 URL  
 指定的資源，在網際網路或近端內部網路的位置是絕對的 URL 或對等的 ADO 連接物件所指定的起始點的相對不完整的 URL。 實際上，串連絕對和相對 Url consitute 是完整的 URL。 另請參閱 URL 和絕對 URL。  
  
 遠端資料來源  
 資料來源存在的另一部電腦上，而不是本機系統上 （用戶端應用程式所執行所在）。  
  
 資源記錄  
 從包含欄位的定義和描述的資料夾或文件之文件來源提供者記錄。 文件本身並未包含在資源記錄，但通常可以使用預設的資料流或包含 URL 的資源記錄中的欄位存取。 另請參閱文件來源提供者，預設資料流，URL。  
  
 資料列集  
 從資料來源，全部都具有相同的欄位結構描述資料列的一組。 資料列集可以代表資料表中的所有或部分欄位。 資料列集也可以表示虛擬資料表，由一個或兩個或多個資料表的聯結。 在 ADO 中，資料列集都由**資料錄集**物件。  
  
## <a name="s"></a>S  
 범위  
 參考的物件或變數或檢視表或資料表中的記錄範圍的範圍。 例如，本機變數可以參考只有在所定義的程序內。 公用變數是可從應用程式的任何位置存取。 物件，例如目前的資料庫，是在範圍內，如果它們是在定義的搜尋路徑。 記錄的範圍可以指定範圍子句中的許多命令。  
  
 服務提供者  
 以服務封裝的產生和取用資料，加強功能，在 ADO 應用程式中的軟體。 它是不會直接公開資料提供者，而是提供服務，例如查詢處理。 服務提供者可能會處理資料提供者所提供的資料。 另請參閱資料提供者。  
  
 形狀的資料錄集  
 A**資料錄集**其資料行已明確定義為包含不只是資料，但是也參考 （稱為章節），其他**資料錄集**物件和/或計算的值根據其他**資料錄集**物件。  
  
 同層級  
 任何兩個或多個節點階層式結構階層架構中位於同一層級。 階層中的根節點有沒有同層級成員。  
  
 預存程序  
 先行編譯程式碼，例如 SQL 陳述式和選擇性的流程控制陳述式的名稱下儲存和處理做為一個單位的集合。 預存程序會儲存在資料庫中。它們可以由應用程式的某個呼叫執行，且允許使用者宣告的變數、 條件式執行和其他功能強大的程式設計功能。  
  
 stub  
 提供參數封送處理的特定介面的物件，而且從不同的執行緒或另一個處理序在不同的執行環境中，執行這類的用戶端接收呼叫的應用程式物件所需的通訊。 虛設常式會位於與應用程式物件，而且會與對應的 proxy 位於與它所呼叫的用戶端進行通訊。 另請參閱 proxy。  
  
 sub-node  
 請參閱子系。  
  
 同步作業  
 作業完成之前可能會啟動下一個作業的程式碼所起始。 另請參閱非同步作業。  
  
## <a name="t-z"></a>T-Z  
 樹狀  
 結構，代表項目 （節點） 之間的階層式關聯性。 沒有最上層 （根） 樹狀結構的一個節點。 下方根目錄中，可以有多個子系。 每個子系又可能是其他子系，因此類似樹分支的父代。 包含文件及其他資料夾的資料夾是樹狀結構的典型範例。 另請參閱階層、 節點、 根、 子系、 父系。  
  
 網頁伺服器  
 提供 Web 服務和內部網路和網際網路使用者頁面的電腦。
