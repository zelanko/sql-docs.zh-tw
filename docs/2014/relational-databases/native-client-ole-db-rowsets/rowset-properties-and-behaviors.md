# <a name="rowset-properties-and-behaviors"></a>資料列集屬性和行為
  這些是[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者資料列集屬性。  
  
|屬性識別碼|描述|  
|-----------------|-----------------|  
|DBPROP_ABORTPRESERVE|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：中止操作之後的資料列集行為取決於此屬性。<br /><br /> VARIANT_FALSE:[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會使中止操作之後無效資料列集。 資料列集物件的功能都會幾乎遺失。 它僅支援 **IUnknown** 作業，以及未處理之資料列與存取子控制代碼的釋放。<br /><br /> VARIANT_TRUE:[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會維持有效的資料列集。|  
|DBPROP_ACCESSORDER|R/W:讀取/寫入<br /><br /> 預設：DBPROPVAL_AO_RANDOM<br /><br /> 描述：存取順序。 必須在資料列集上存取資料行的順序。<br /><br /> DBPROPVAL_AO_RANDOM:可以依任何順序存取資料行。<br /><br /> DBPROPVAL_AO_SEQUENTIALSTORAGEOBJECTS:資料行繫結為儲存體物件只能由資料行序數決定的循序存取。<br /><br /> DBPROPVAL_AO_SEQUENTIAL:必須由資料行序數決定的循序存取所有的資料行。|  
|DBPROP_APPENDONLY|這個資料列集屬性不會實作[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者。 嘗試讀取或寫入屬性值會產生錯誤。|  
|DBPROP_BLOCKINGSTORAGEOBJECTS|R/W:唯讀<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者儲存物件區塊使用其他資料列集的方法。|  
|DBPROP_BOOKMARKS DBPROP_LITERALBOOKMARKS|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援資料列集資料列識別書籤時將 DBPROP_BOOKMARKS 或 DBPROP_LITERALBOOKMARKS 為 VARIANT_TRUE。<br /><br /> 將任一個屬性設定為 VARIANT_TRUE 不會依書籤啟用資料列集定位。 將 DBPROP_IRowsetLocate 或 DBPROP_IRowsetScroll 設定為 VARIANT_TRUE 來建立支援依書籤進行資料列集定位的資料列集。<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會使用[!INCLUDE[msCoName](../../includes/msconame-md.md)][!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]資料指標來支援包含書籤的資料列集。 如需詳細資訊，請參閱[資料列集和 SQL Server 資料指標](rowsets-and-sql-server-cursors.md)。 **注意：** 設定這些屬性與其他衝突[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者資料指標定義屬性會導致錯誤。 例如，當 DBPROP_OTHERINSERT 也為 VARIANT_TRUE 時，將 DBPROP_BOOKMARKS 設定為 VARIANT_TRUE 會在取用者嘗試開啟資料列集時產生錯誤。|  
|DBPROP_BOOKMARKSKIPPED|R/W:唯讀<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]如果取用者無效的書籤定位或搜尋加上書籤的資料列集時，Native Client OLE DB 提供者會傳回 DB_E_BADBOOKMARK。|  
|DBPROP_BOOKMARKTYPE|R/W:唯讀<br /><br /> 預設：DBPROPVAL_BMK_NUMERIC<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會實作數值書籤。 A [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者書籤是 32 位元不帶正負號的整數，類型為 DBTYPE_UI4。|  
|DBPROP_CACHEDEFERRED|這個資料列集屬性不會實作[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者。 嘗試讀取或寫入屬性值會產生錯誤。|  
|DBPROP_CANFETCHBACKWARDS DBPROP_CANSCROLLBACKWARDS|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者支援反向提取和捲動在裝置不按順序的資料列集。 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]當 DBPROP_CANFETCHBACKWARDS 或 DBPROP_CANSCROLLBACKWARDS 為 VARIANT_TRUE 時，Native Client OLE DB 提供者會建立資料指標支援資料列集。 如需詳細資訊，請參閱[資料列集和 SQL Server 資料指標](rowsets-and-sql-server-cursors.md)。|  
|DBPROP_CANHOLDROWS|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：根據預設， [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會傳回 DB_E_ROWSNOTRELEASED 如果取用者會嘗試取得更多資料列的資料列集時暫止的變更存在於目前在資料列集。 這個行為可以修改。<br /><br /> 將 DBPROP_CANHOLDROWS 和 DBPROP_IRowsetChange 同時設定為 VARIANT_TRUE 意味者加上書籤的資料列集。 如果兩個屬性都為 VARIANT_TRUE，在資料列集上會提供 **IRowsetLocate** 介面，而且 DBPROP_BOOKMARKS 和 DBPROP_LITERALBOOKMARKS 都為 VARIANT_TRUE。<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者的資料列集包含書籤都受到[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]資料指標。|  
|DBPROP_CHANGEINSERTEDROWS|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：這個屬性只能設定為 VARIANT_TRUE，如果資料列集使用索引鍵集驅動資料指標。|  
|DBPROP_COLUMNRESTRICT|R/W:唯讀<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者將屬性設定為 VARIANT_TRUE 時，取用者無法變更資料列集中的資料行。 資料列集中的其他資料行可能可以更新，而且資料列本身可能會遭到刪除。<br /><br /> 當屬性為 VARIANT_TRUE 時，取用者會檢查 DBCOLUMNINFO 結構的 *dwFlags* 成員來判斷是否可寫入個別資料行的值。 對於可修改的資料行，*dwFlags* 會表現 DBCOLUMNFLAGS_WRITE。|  
|DBPROP_COMMANDTIMEOUT|R/W:讀取/寫入<br /><br /> 預設：0<br /><br /> 描述：根據預設， [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者不會逾時上**icommand:: Execute**方法。|  
|DBPROP_COMMITPRESERVE|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：認可作業之後的資料列集行為取決於此屬性。<br /><br /> VARIANT_TRUE:[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會維持有效的資料列集。<br /><br /> VARIANT_FALSE:[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會使認可作業之後無效資料列集。 資料列集物件的功能都會幾乎遺失。 它僅支援 **IUnknown** 作業，以及未處理之資料列與存取子控制代碼的釋放。|  
|DBPROP_DEFERRED|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：當設定為 VARIANT_TRUE [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會嘗試使用伺服器資料指標的資料列集。 在應用程式存取 **Text**、**ntext** 和 **image** 資料行之前，不會從伺服器傳回這些資料行。|  
|DBPROP_DELAYSTORAGEOBJECTS|R/W:唯讀<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者儲存物件上支援立即更新模式。<br /><br /> 對循序資料流物件中的資料所進行之變更會立即提交至 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]。 系統會根據資料列集交易模式認可這些修改。|  
|DBPROP_HIDDENCOLUMNS|R/W:唯讀<br /><br /> 預設：VARIANT_FALSE<br /><br /> **描述：** 隱藏資料行計數<br /><br /> 如果 DBPROP_UNIQUEROWS 為 VARIANT_TRUE，DBPROP_HIDDENCOLUMNS 屬性會傳回提供者所加入之其他「隱藏」資料行的數目，藉以唯一識別資料列集中的資料列。 這些資料行會由 **IColumnsInfo::GetColumnInfo** 和 **IColumnsRowset::GetColumnsRowset** 傳回。 不過，這些資料行不會包含在 **IColumnsInfo::GetColumnInfo** 所傳回之 *pcColumns* 引數傳回的資料列計數中。<br /><br /> 為判斷 **IColumnsInfo::GetColumnInfo** 所傳回之 *prgInfo* 結構中表示的資料行總數，包括隱藏的資料行，取用者會將 DBPROP_HIDDENCOLUMNS 的值加入到從 *pcColumns* 之 **IColumnsInfo::GetColumnInfo** 中所傳回的資料行計數。 如果 DBPROP_UNIQUEROWS 為 VARIANT_FALSE，DBPROP_HIDDENCOLUMNS 為零。|  
|DBPROP_IAccessor DBPROP_IColumnsInfo DBPROP_IConvertType DBPROP_IRowset DBPROP_IRowsetInfo|R/W:唯讀<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援這些介面在所有資料列集。|  
|DBPROP_IColumnsRowset|R/W:讀取/寫入<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援**IColumnsRowset**介面。|  
|DBPROP_IConnectionPointContainer|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：IConnectionPointContainer。 如果是 VARIANT_TRUE，資料列集支援指定的介面。 如果是 VARIANT_FALSE，資料列集不支援指定的介面。 支援介面的提供者必須支援與包含 VARIANT_TRUE 值之介面相關聯的屬性。 這些屬性主要用於透過 ICommandProperties::SetProperties 來要求介面。|  
|DBPROP_IMultipleResults|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援**IMultipleResults**介面。|  
|DBPROP_IRowsetChange DBPROP_IRowsetUpdate|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援**IRowsetChange**並**IRowsetUpdate**介面。<br /><br /> 使用 DBPROP_IRowsetChange 等於 VARIANT_TRUE 建立的資料列集會表現立即更新模式行為。<br /><br /> 當 DBPROP_IRowsetUpdate 是 VARIANT_TRUE 時，DBPROP_IRowsetChange 也是 VARIANT_TRUE。 資料列集會表現延遲的更新模式行為。<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會使用[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]資料指標來支援資料列集公開**IRowsetChange**或是**IRowsetUpdate**。 如需詳細資訊，請參閱[資料列集和 SQL Server 資料指標](rowsets-and-sql-server-cursors.md)。|  
|DBPROP_IRowsetIdentity|R/W:讀取/寫入<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會支援**IRowsetIdentity**介面。 如果資料列集支援此介面，代表相同基礎資料列的任兩個資料列控制代碼永遠會反映相同的資料和狀態。 取用者可以呼叫**IRowsetIdentity::IsSameRow**方法來比較兩個資料列控制代碼，以查看它們是否參考相同的資料列執行個體。|  
|DBPROP_IRowsetLocate DBPROP_IRowsetScroll|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者可以公開**IRowsetLocate**並**IRowsetScroll**介面。<br /><br /> 當 DBPROP_IRowsetLocate 是 VARIANT_TRUE 時，DBPROP_CANFETCHBACKWARDS 和 DBPROP_CANSCROLLBACKWARDS 也是 VARIANT_TRUE。<br /><br /> 如果 DBPROP_IRowsetScroll 為 VARIANT_TRUE，DBPROP_IRowsetLocate 也為 VARIANT_TRUE，而且在資料列集上會提供這兩個介面。<br /><br /> 這兩種介面都需要書籤。 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會設定將 DBPROP_BOOKMARKS 和 DBPROP_LITERALBOOKMARKS 為 VARIANT_TRUE 時取用者要求任一種介面。<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會使用[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]資料指標來支援**IRowsetLocate**並**IRowsetScroll**。 如需詳細資訊，請參閱[資料列集和 SQL Server 資料指標](rowsets-and-sql-server-cursors.md)。<br /><br /> 設定這些屬性與其他衝突[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者資料指標定義屬性會導致錯誤。 例如，當 DBPROP_OTHERINSERT 也為 VARIANT_TRUE 時，將 DBPROP_IRowsetScroll 設定為 VARIANT_TRUE 會在取用者嘗試開啟資料列集時產生錯誤。|  
|DBPROP_IRowsetResynch|R/W:讀取/寫入<br /><br /> 預設：VARIANT_FALSE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會公開**IRowsetResynch**隨選的介面。 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者可以公開任何資料列集上的介面。|  
|DBPROP_ISupportErrorInfo|R/W:讀取/寫入<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB 提供者會公開**ISupportErrorInfo**上資料列集介面。|  
|DBPROP_ILockBytes|不會實作這個介面[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者。 嘗試讀取或寫入屬性會產生錯誤。|

|DBPROP_IStream|不會實作這個介面[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Native Client OLE DB 提供者。 嘗試讀取或寫入屬性會產生錯誤。|  
|DBPROP_IMMOBILEROWS|R/W:讀取/寫入<br /><br /> 預設：VARIANT_TRUE<br /><br /> 描述：屬性是僅為 VARIANT_TRUE 的[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]keyset 資料指標; 它的所有其他資料指標為 VARIANT_FALSE。<br /><br /> VARIANT_TRUE:資料列集不會重新排列插入或更新的資料列。 對於 **IRowsetChange::InsertRow**，資料列將會出現在資料列集的結尾。 對於 **IRowsetChange::SetData**，如果資料列集未經過排序，則不會變更更新之資料列的位置。 如果資料列集經過排序，而且 **IRowsetChange::SetData** 變更用於排序資料列集的資料行，則不會移動資料列。 如果資料列集是在一組索引鍵資料行 (通常是 DBPROP_OTHERUPDATEDELETE 為 VARIANT_TRUE，但是 DBPROP_OTHERINSERT 為 VARIANT_FALSE 的資料列集) 上建立的，變更索引鍵資料行的值通常相當於刪除目前的資料列並插入新的資料列。 因此，如果 DBPROP_OWNINSERT 為 VARIANT_FALSE，即使 DBPROP_IMMOBILEROWS 屬性為 VARIANT_TRUE，資料列可能會像是移動，或甚至從資料列集消失。<br /><br /> VARIANT_FALSE:如果資料列集經過排序，插入的資料列會出現在資料列集的正確的順序。 如果資料列集未經過排序，插入的資料列會出現在結尾。 如果 **IRowsetChange::SetData** 變更用於排序資料列集的資料行，則不會移動資料列。 如果資料列集未經過排序，則不會變更資料列的位置。|
