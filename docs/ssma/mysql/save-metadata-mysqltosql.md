# <a name="save-metadata--mysqltosql"></a>儲存中繼資料 (MySQLToSQL)
[**儲存中繼資料**] 對話方塊會提示您將中繼資料載入至 SSMA 專案，然後再加以儲存。 這可讓您擁有可離線使用並傳送給其他人（例如技術支援人員）的完整專案檔案。  
  
若要存取 [**儲存中繼資料**] 對話方塊，請儲存專案。 如果遺漏任何中繼資料，SSMA 將會顯示 [**儲存中繼資料**] 對話方塊。  
  
## <a name="options"></a>選項。  
**名稱**  
專案中每個資料庫的名稱。  
  
**狀態**  
指出中繼資料是否已載入至 SSMA 專案，或中繼資料是否遺失。  
  
SSMA 會視需要將中繼資料載入專案。 當您流覽中繼資料和轉換架構時，會自動載入中繼資料。  
  
**全選**  
選取所有列出的資料庫。  
  
**清除**  
清除所有遺失中繼資料之資料庫的核取方塊。 如果已載入中繼資料，則無法清除此核取方塊。  
  
**儲存**  
儲存專案，並載入具有遺漏中繼資料之所選資料庫的中繼資料。  
  
**取消**  
取消儲存作業。 遺漏中繼資料不會載入專案。
