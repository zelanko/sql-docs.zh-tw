# <a name="o2ss0331-unable-to-convert-fetch-statement-error"></a>O2SS0331：無法轉換 FETCH 語句（錯誤）

本文描述在以參數傳回的`FETCH ... INTO` `SYS_REFCURSOR`程式中轉換語句時，ORACLE 的 SQL Server 移轉小幫手（SSMA）為何會產生錯誤。

## <a name="background"></a>背景

在 Oracle 中`SYS_REFCURSOR` ，是用來將和的資料指標傳遞給預存程式。

每當 SSMA 遇到一個變數做`SYS_REFCURSOR`為程式的傳回型別，並使用相同的變數搭配`FETCH ... INTO`語句時，就會產生錯誤訊息。

## <a name="example"></a>範例

請考慮下列範例查詢，其中我們已將變數宣告為`SYS_REFCURSOR`，做為程式的傳回型別使用。

```sql
CREATE OR REPLACE PROCEDURE p_close_refcursor
(
    emp_refcur OUT SYS_REFCURSOR
)
IS
    departmentno dept.deptno%TYPE;
BEGIN
    OPEN
        emp_refcur
    FOR
        SELECT deptno
        FROM dept;

    LOOP
        FETCH emp_refcur
        INTO departmentno;

        EXIT WHEN emp_refcur%NOTFOUND;

        DBMS_OUTPUT.PUT_LINE(departmentno);
    END LOOP;

    CLOSE emp_refcur;
END;
```

當您嘗試在 SSMA 中轉換上述程式碼時，它會產生下列錯誤訊息：

> O2SS0330：無法轉換 FETCH 語句

## <a name="possible-remedies"></a>可能的解決辦法

修正此錯誤有兩種可能的補救方式。

### <a name="use-local-cursor-variable"></a>使用本機資料指標變數

第一個方法是建立並使用本機資料指標來執行所有作業，然後將其值傳遞至傳回的資料指標，然後再將它關閉。 為此，請宣告`SYS_REFCURSOR`類型的新變數，這將會在此程式的本機。 修改程式碼，以使用此本機資料指標來執行必要的`FETCH ... INTO`作業，包括語句。 在關閉本機資料指標之前，請將這個本機資料指標的值傳遞至程式的參數區段中所定義`emp_refcur`的資料指標（在我們的範例中為）。

上述錯誤的解決方案會顯示在下列程式碼中：

```sql
CREATE OR REPLACE PROCEDURE p_close_refcursor
(
    emp_refcur OUT SYS_REFCURSOR
)
AS
    test_cursor SYS_REFCURSOR;
    departmentno dept.deptno%TYPE;
BEGIN
    OPEN
        test_cursor
    FOR
        SELECT deptno
        FROM dept;

    LOOP
        FETCH test_cursor
        INTO departmentno;

        EXIT WHEN test_cursor%NOTFOUND;

        DBMS_OUTPUT.PUT_LINE(departmentno);
    END LOOP;

    emp_refcur := test_cursor;

    CLOSE test_cursor;
END;
```

### <a name="return-result-set"></a>傳回結果集

解決這個錯誤的另一種方法是使用自然的方法，然後在 SQL Server，也就是直接從執行預存程式傳回結果集。 這種方法會在[O2SS0157](o2ss0157.md)轉換訊息一文的詳細資料中說明。
