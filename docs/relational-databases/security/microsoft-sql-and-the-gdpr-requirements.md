# <a name="guide-to-enhancing-privacy-and-addressing-gdpr-requirements-with-the-microsoft-sql-platform"></a>增強的隱私權和定址 GDPR 需求與 Microsoft SQL 平台指南


## <a name="summary"></a>摘要
歐洲的隱私權法律 2018 月 25，是因為設定隱私權權限、 安全性與相容性的新全域列就會生效。 一般資料保護規定或 GDPR，是本質上關於保護，以及啟用隱私權權限的個人、，以及建立控管如何個人資料管理，且受保護，同時尊重個別選擇嚴格通用的隱私權需求。 

受限於 GDPR 是否管理雲端架構的 Microsoft SQL 客戶或內部部署資料庫或兩者都必須確保資料庫系統中限定資料是相似處理及保護根據 GDPR 原則。 這表示，許多客戶需要檢閱或修改其資料庫管理和資料處理程序，尤其將焦點放在資料處理為 GDPR 中約定的安全性。

Microsoft SQL 為基礎的技術提供許多內建安全性功能，可協助資料來降低風險並改善其保護和管理性的資料庫層級和更新版本的資料。 這份文件會檢查這些功能，並共用一些使用 Microsoft SQL 針對達到資料隱私權的目標 GDPR Microsoft 自己的方法。
