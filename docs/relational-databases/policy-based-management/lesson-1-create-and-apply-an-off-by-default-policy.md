# <a name="lesson-1-create-and-apply-an-off-by-default-policy"></a>第 1 課：建立和套用 Off By Default 原則
[!INCLUDE[appliesto-ss-xxxx-xxxx-xxx-md](../../includes/appliesto-ss-xxxx-xxxx-xxx-md.md)] 您可以使用原則式管理原則來管理一或多個 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 執行個體、一或多個執行個體物件、伺服器執行個體、一或多個資料庫，或是一或多個資料庫物件。 身為資料庫管理員，您想要確保特定伺服器沒有啟用 Database Mail。 在這一課，您將建立一個條件，以及設定這個伺服器選項的原則。 您將會測試伺服器，以便查看它是否符合此原則。 然後，您將會使用此原則來重新設定伺服器，以便讓伺服器符合。  
  
這個課程包含下列主題：  
  
-   [建立依預設為關閉的原則](../../relational-databases/policy-based-management/lesson-1-1-create-the-off-by-default-policy.md)  
  
-   [將伺服器設定為執行 Off By Default 原則](../../relational-databases/policy-based-management/lesson-1-2-configure-a-server-to-run-the-off-by-default-policy.md)
